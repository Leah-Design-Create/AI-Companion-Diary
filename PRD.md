**文档版本**：v1.0  
**更新日期**：2025-03-13  
**产品名称**：树洞日记（小伴）

---

## 一、产品概述

### 1.1 产品定位

「树洞日记」是一款以**情感陪伴**为核心的情感陪伴类 AI 产品。以「树洞」为形象，为用户提供日常倾诉、情绪倾听、生活记录与心理支持。产品融合**意图识别**与 **RAG 知识库**，可引用用户上传的书籍或文章内容，在陪伴聊天中提供有依据的心理/情绪类知识支持。

### 1.2 产品愿景

打造一个温暖、私密、可信任的情感陪伴空间，让用户能够：
- 随时倾诉，获得共情式回应
- 记录生活与情绪，形成可回顾的日记
- 在需要时获得基于个人知识库的专业参考
- 感受到持续、主动的关心（如久未登录时的问候）

### 1.3 核心价值主张

| 价值点 | 说明 |
|--------|------|
| **温暖陪伴** | 口语化、不评判、多共情，像朋友聊天 |
| **私密树洞** | 多会话管理，不同话题可分开记录 |
| **智能知识** | 意图识别 + RAG，自动引用用户上传的参考内容 |
| **情绪洞察** | 当日心情、单次心情、焦虑标记，让用户更了解自己 |
| **生活记录** | 每次对话可生成简短总结，形成可回顾的日记 |

## 二、目标用户

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| **主要人群** | 有情感倾诉需求、希望获得情绪支持的成年人 |
| **使用场景** | 日常倾诉、压力释放、情绪梳理、睡前聊天、记录生活 |
| **核心诉求** | 被倾听、被理解、获得共情、偶尔需要专业参考 |
| **技术门槛** | 能使用浏览器访问网页即可，无需安装 App |

### 2.2 用户故事

- 作为用户，我希望有一个可以随时倾诉的对象，不被评判。
- 作为用户，我希望对话能被记录下来，形成可回顾的生活日记。
- 作为用户，当我问及焦虑、情绪等问题时，希望得到基于我上传书籍/文章的回答。
- 作为用户，我希望了解自己当天的情绪状态，以及每次对话时的心情。
- 作为用户，当我很久没来时，希望收到树洞的问候，感受到被惦记。

## 三、功能需求

### 3.1 功能架构图

```
树洞日记
├── 日常陪聊
│   ├── 文字对话
│   ├── 图片分享
│   ├── 语音输入（浏览器）
│   └── 回复朗读（浏览器 TTS / DashScope TTS）
├── 多会话管理
│   ├── 新建对话
│   ├── 重命名
│   ├── 删除
│   └── 切换会话继续上次对话
├── 意图识别 + RAG
│   ├── 闲聊 vs 问知识库
│   ├── 关键词 / 语义检索
│   └── 自动引用（用户无感知）
├── 情绪与日记
│   ├── 当日心情（聊天页右上角）
│   ├── 单次心情（日记卡片）
│   ├── 焦虑分析（会话结束）
│   └── 生活记录（总结 + 日历 + 详情）
└── 久未登录提醒
    └── 超过 N 天未访问时主动问候
```

### 3.2 功能详细说明

#### 3.2.1 日常陪聊

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 文字对话 | 用户输入文字，树洞以温暖、口语化方式回复；支持多轮对话 | P0 |
| 图片分享 | 支持上传 jpg/png/gif/webp，多模态模型可看图回复 | P0 |
| 语音输入 | 支持浏览器语音输入（Web Speech API） | P1 |
| 回复朗读 | 点击「朗读」按钮，使用浏览器 TTS 或 DashScope 高品质 TTS | P1 |

**交互规则**：
- 回复简短、口语化，避免像念稿子
- 共情方式多样，不固定套路
- 表情符号偶尔使用，不每条都用

#### 3.2.2 多会话管理

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 会话列表 | 左侧栏展示历史话题，按开始时间倒序 | P0 |
| 新建对话 | 点击「新建」创建新会话 | P0 |
| 重命名 | 支持为会话设置自定义标题 | P1 |
| 删除 | 支持删除会话及全部消息 | P1 |
| 切换会话 | 点击某会话可加载历史消息并继续对话 | P0 |

#### 3.2.3 意图识别 + RAG 知识库

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 意图识别 | 自动判断「闲聊」或「问知识库」 | P0 |
| 知识库检索 | 问知识库时从后台检索用户上传的文档 | P0 |
| 检索方式 | 支持关键词检索；配置 Embedding 后支持语义检索 | P0 |
| 引用方式 | 检索到的内容注入模型上下文，用户无感知，回复自然 | P0 |
| 追问跳过 | 「详细说说」「然后呢」等短追问不触发 RAG，避免误注入 | P0 |
| 知识库管理 | 管理员通过 /upload 页面上传 .txt / .pdf | P0 |

**意图判定规则**：
- 问知识库：涉及焦虑、情绪、心理、压力、书中内容等
- 闲聊：打招呼、聊生活/爱好、与文档无关的话题

#### 3.2.4 焦虑分析

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 触发时机 | 用户点击「结束会话」时 | P0 |
| 分析内容 | 判断对话中是否存在明显焦虑、压力或负面情绪 | P0 |
| 标记存储 | 在会话上标记 `anxiety_detected` | P0 |
| 后续影响 | 后续回复更偏共情与支持 | P0 |

#### 3.2.5 心情识别

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 当日心情 | 按当天所有对话综合计算，展示在聊天页右上角 | P0 |
| 单次心情 | 每条日记卡片右下角显示该次对话的当时心情 | P0 |
| 心情词库 | 如：平静、开心、焦虑、难过、期待、疲惫、困惑、放松等 | P0 |
| 补全机制 | 旧记录无 mood 时，接口可尝试补全 | P1 |

#### 3.2.6 生活记录（日记）

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 总结生成 | 会话结束后生成 100–200 字简短总结，第二人称「你」 | P0 |
| 日记列表 | 右侧日记页展示历史总结，含心情、图片 | P0 |
| 日历 | 右侧有日历，点击有记录的日期可定位 | P0 |
| 查看详情 | 点击「查看详情」打开该次完整聊天记录 | P0 |
| 图片展示 | 日记卡片可展示该次对话中用户分享的图片 | P1 |

#### 3.2.7 久未登录提醒

| 需求项 | 说明 | 优先级 |
|--------|------|--------|
| 触发条件 | 超过配置天数（默认 2 天）未访问 | P0 |
| 问候内容 | 以小伴身份写简短问候或分享 | P0 |
| 调用时机 | 用户打开应用时调用 `/api/check-in` | P0 |

## 四、产品界面与交互

### 4.1 整体布局

- **左侧**：会话列表（可折叠），含新建、重命名、删除
- **中间**：聊天区域，输入框、发送、图片、语音、朗读
- **右侧**：日记页，含日历、生活记录列表、详情弹窗

### 4.2 设计风格

- 树洞日记风格：温暖、私密、手账感
- 配色：暖色系（`#FAF7F2`、`#F3EDE5`、`#FCE6B4` 等）
- 字体：Noto Serif SC、Noto Sans SC

### 4.3 关键交互流程

1. **首次打开** → 调用 check-in → 若久未登录则展示问候 → 进入聊天
2. **发送消息** → 意图识别 → 若问知识库则 RAG 检索 → 生成回复 → 更新当日心情
3. **结束会话** → 焦虑分析 + 总结 + 心情 → 写入日记
4. **查看日记** → 日历选日期 → 列表定位 → 点击详情查看完整对话

## 五、技术架构

### 5.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 后端 | Python 3.10+、FastAPI、aiosqlite |
| 前端 | 单页 HTML + 原生 JS，无框架 |
| 数据库 | SQLite |
| LLM | OpenAI 兼容 API（DeepSeek、通义、Kimi、智谱等） |
| 向量 | 可选 Embedding 模型（语义检索） |
| TTS | 浏览器 TTS / DashScope TTS（可选） |

### 5.2 数据模型

| 表名 | 说明 |
|------|------|
| users | 用户（id, name, last_login_at） |
| sessions | 会话（id, user_id, started_at, ended_at, summary, anxiety_detected, mood, title） |
| messages | 消息（id, session_id, role, content, image_path） |
| knowledge | 知识库（id, title, content, source_url, embedding） |
| reminders | 久未登录问候记录（可选） |

### 5.3 API 概览

| 分类 | 接口 | 说明 |
|------|------|------|
| 入口 | GET / | 前端页面 |
| 签到 | GET /api/check-in | 久未登录提醒 |
| 心情 | GET /api/mood | 当日综合心情 |
| 对话 | POST /api/chat | 发送文字 |
| 对话 | POST /api/chat/send | 发送文字+图片 |
| 对话 | POST /api/chat/stream | 流式回复 |
| 会话 | POST /api/session/end | 结束会话 |
| 会话 | GET /api/sessions | 会话列表 |
| 会话 | GET /api/sessions/{id}/messages | 会话消息 |
| 会话 | PATCH /api/sessions/{id} | 重命名 |
| 会话 | DELETE /api/sessions/{id} | 删除 |
| 日记 | GET /api/summaries | 生活记录列表 |
| 知识库 | GET /upload | 上传页面 |
| 知识库 | POST /api/knowledge/upload | 上传文件 |
| 其他 | POST /api/tts | 文本转语音 |

## 六、非功能需求

### 6.1 性能

- 单次对话响应时间：目标 < 10s（取决于 LLM 服务）
- 会话列表加载：< 500ms
- 知识库检索：< 2s（语义检索取决于 Embedding 服务）

### 6.2 安全与隐私

- 用户数据本地存储（SQLite），不上传至第三方（除 LLM/Embedding API）
- 图片仅存于本地 uploads 目录
- 知识库上传页不对外展示，仅管理员使用

### 6.3 兼容性

- 支持 Chrome、Edge、Safari、Firefox 等现代浏览器
- 多模态需使用支持视觉的模型（如 gpt-4o、qwen-vl、glm-4v）

### 6.4 可配置项

| 配置项 | 说明 | 默认 |
|--------|------|------|
| OPENAI_API_KEY | LLM API Key | 必填 |
| OPENAI_BASE_URL | API 基地址 | https://api.openai.com/v1 |
| OPENAI_MODEL | 模型名 | gpt-4o-mini |
| OPENAI_EMBEDDING_MODEL | 嵌入模型 | 空则仅关键词 |
| DB_PATH | 数据库路径 | companion.db |
| INACTIVE_DAYS_FOR_REMINDER | 久未登录天数 | 2 |
| DASHSCOPE_* | TTS 相关 | 空则用浏览器朗读 |
